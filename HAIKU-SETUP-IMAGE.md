# Haiku 捷徑改造：從上傳文字改為上傳圖片

## 改動概述

**舊流程：**
```
截圖 → OCR 掃文字 → 上傳文字 → AI 讀文字
```

**新流程：**
```
截圖 → 轉 base64 圖片 → 上傳圖片 → AI 直接看圖識圖
```

好處：AI 能直接看圖，識圖效果更好，不用先 OCR 文字。

---

## Haiku 捷徑改法

### 1. 刪除「從最新的照片 擷取文字」這一步

因為不再需要 OCR，直接用圖片。

### 2. 改「要求內文」的設置

**改前：**
```json
{
  "text": "掃描出來的文字"
}
```

**改後：**
```json
{
  "image": "base64 圖片字串"
}
```

---

## 具體步驟

1. **打開「取得 URL 內容」這個 action**

2. **刪掉「要求內文」裡的 `text` 欄位**（點紅色 ❌）

3. **加新欄位：**
   - 點「**加入新欄位**」
   - Key 填：`image`
   - Value：需要填「圖片轉 base64 的值」

4. **問題：怎麼把圖片轉 base64？**

   在「從最新的照片 擷取文字」這一步**之前**，加一個新的 action：
   
   ```
   「取得圖片數據」或「編碼」
   ```
   
   （Haiku 裡應該有現成的 action 把圖片轉 base64）
   
   或者用一個簡單的方式：
   - 用「快捷指令」內建的「取得變數」
   - 找「圖片 → base64」的功能

5. **確認最新的照片變數能傳給這個 base64 action**

---

## 簡化方案（如果 Haiku 沒有現成的 base64 action）

直接在「要求內文」用「圖片」類型而不是「JSON」：

1. 把「要求內文」改成「**二進制**」或「**圖片**」類型
2. 連接「最新的照片」變數

Haiku 會自動把圖片轉成 base64 並發送。

## 後端已改好

- `screenshotStore-improved.js` - 保存圖片
- `server-with-image.js` - 接收圖片、返回圖片給 AI

部署步驟：
1. 把 `server-with-image.js` 改名為 `server.js`
2. 把 `screenshotStore-improved.js` 改名為 `screenshotStore.js`
3. 把這兩個檔案上傳到 GitHub
4. Render 會自動重新部署
